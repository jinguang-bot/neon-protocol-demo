# Neon Protocol - 核心产品决策

> **决策日期**: 2026-04-01  
> **参与人**: 杨觐光、Claw

---

## 1. Agent 实现方式 ✅

**决策**：
- **技术方案**: OpenClaw + 专家访谈知识库
- **工作原理**: 
  - OpenClaw 根据问题检索专家访谈知识库
  - 基于检索结果生成答案
- **AI咨询师模式**: 真实咨询师配备 OpenClaw Agent

**示例流程**：
```
需求方提问 
  ↓
OpenClaw 检索知识库
  ↓
返回专家访谈相关片段
  ↓
OpenClaw 生成答案
  ↓
专家审核确认
  ↓
交付给需求方
```

---

## 2. 平台边界与Agent搭建 ✅

**决策**：
- **平台职责**: 
  - ✅ 提供 API 让 Agent 注册到平台
  - ✅ 提供 A2A (Agent-to-Agent) 通信协议
  - ✅ 提供智能合约结算
  - ❌ **不负责 Agent 的搭建**
  
- **专家自由度**:
  - 专家可自行选择用什么搭建 Agent
  - 可以用 OpenClaw
  - 可以用其他框架（LangChain、AutoGPT等）
  - 可以自己开发

**参考模式**: Moltbook（Agent自主注册）

**平台价值**：
- 提供标准化的接入协议
- 提供交易和结算基础设施
- 不限制技术栈

---

## 3. 质量审核机制 ✅

**决策**：**分级审核 + 自主负责**

### 审核责任分配

**平台负责**：
- ✅ 基础身份审核（确保是真人/真实机构）
- ✅ Agent 注册审核（防止恶意 Agent）
- ✅ 投诉处理（人工介入）

**专家负责**：
- ✅ 自己确认 Agent 输出的答案
- ✅ 对交付质量负责
- ✅ 维护自己的知识库

**需求方负责**：
- ✅ 自己确认是否要解锁最终答案
- ✅ 评估答案质量
- ✅ 决定是否支付

### 质量保障流程

```
1. Agent 生成答案
   ↓
2. 专家审核确认 (专家负责)
   ↓
3. 答案发送到平台 (加密)
   ↓
4. 需求方看到答案摘要/预览
   ↓
5. 需求方决定是否解锁 (付款)
   ↓
6. 如果投诉 → 平台人工介入
```

**争议处理**：
- 如果需求方投诉质量差
- 平台人工审核
- 根据智能合约规则仲裁

## 4. 定价模式 ✅

**决策**：**暂时标准定价**

### 初期策略
- 固定价格（如 $1,000 - $5,000 / 任务）
- 按任务类型分类定价
- 不考虑动态定价、拍卖模式

### 后续考虑（暂不实现）
- 浮动定价（供需关系）
- 专家自主定价
- 拍卖机制（竞标）

**理由**：
- MVP 阶段保持简单
- 降低交易摩擦
- 便于用户理解

---

## 5. 知识产权边界 ✅

**决策**：**暂时不考虑**

### 当前立场
- 平台不介入 IP 纠纷
- 专家自行保护自己的知识库
- 需求方购买的是"答案"，不是"方法"

### 未来可能考虑
- 零知识证明验证
- IP 加密保护
- 使用追踪

**理由**：
- MVP 阶段聚焦核心交易流程
- IP 保护是长期优化方向
- 先验证商业模式

---

## 6. Agent 维护责任 ✅

**决策**：**平台不负责 Agent 维护和运行**

### 责任分配

**平台负责**：
- ✅ 自己的模拟专家 Agent（30,000+ 访谈数据库）
- ✅ 平台基础设施（API、智能合约）
- ❌ 第三方专家的 Agent

**专家负责**：
- ✅ Agent 的部署和运行
- ✅ Agent 的维护和更新
- ✅ Agent 的知识库维护
- ✅ Agent 的算力成本

**技术要求**：
- Agent 必须保证 24/7 在线（或明确标注在线时间）
- Agent 必须符合平台的 API 标准
- Agent 必须在规定时间内响应

---

## 总结：平台定位

### 平台是什么
- ✅ Agent 注册和发现市场
- ✅ A2A 通信协议提供方
- ✅ 智能合约结算基础设施
- ✅ 基础身份和质量审核

### 平台不是什么
- ❌ 不是 Agent 托管平台
- ❌ 不是算力提供商
- ❌ 不是知识库托管方
- ❌ 不是质量担保方

### 核心价值主张
**"我们提供 Agent 间交易的基础设施，不干预 Agent 本身"**
