## 🏗️ 技术栈

### 前端
- **框架**: Next.js 15 (App Router)
- **语言**: TypeScript 5.x
- **样式**: Tailwind CSS 3.x
- **动画**: Framer Motion
- **表单**: React Hook Form + Zod
- **状态**: Zustand / React Context
- **HTTP**: Axios / Fetch

### 后端
- **运行时**: Node.js 18+
- **框架**: Next.js API Routes
- **ORM**: Prisma
- **数据库**: SQLite (dev) / PostgreSQL (prod)
- **验证**: Zod
- **认证**: NextAuth.js (可选)

### 测试
- **单元测试**: Jest + Testing Library
- **E2E 测试**: Playwright
- **API 测试**: Postman / curl

### 部署
- **平台**: Vercel / Netlify
- **数据库**: Vercel Postgres / Supabase
- **文件存储**: Vercel Blob / AWS S3

## 🚨 风险与缓解

### 风险 1: 时间预估过于乐观
- **缓解**: 每个特性预留 20% buffer
- **应对**: 如果延迟，优先完成核心闭环（F001-F011）

### 风险 2: 技术难点（区块链集成）
- **缓解**: Phase 1 使用模拟支付，延后区块链集成
- **应对**: 如果区块链延迟，继续使用平台托管

### 风险 3: 匹配算法效果不佳
- **缓解**: Phase 1 使用简单规则匹配 + 人工调整
- **应对**: 如果匹配不佳，增加人工运营
