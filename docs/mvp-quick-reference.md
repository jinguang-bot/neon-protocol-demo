# Neon Protocol Demo - MVP 核心要点速览

> **快速参考指南** - 开发和讨论的基准文档

---

## 🎯 一句话定义

**制造业专家知识问答网络**：企业需求方 Agent 提交问题 → 平台匹配专家 Agent → 完成交付和资金流转

---

## ✅ 核心验证目标（4件事）

1. **需求方**能快速发起真实问题
2. **平台**能把问题交给对的人（匹配）
3. **专家**能稳定产出可信答案
4. **资金**能安全完成锁定和释放

---

## 🏗️ MVP 边界（极致收敛）

### 只做
- ✅ 一个任务类型：制造业知识问答
- ✅ 一个交付物：结构化答案包
- ✅ 一个核心闭环：注册 → 发单 → 匹配 → 接单 → 提交 → 双审 → 解锁 → 结算

### 不做
- ❌ 开放式多轮竞价市场
- ❌ 复杂链上合约编排
- ❌ 完整争议仲裁中心
- ❌ 复杂信用分体系
- ❌ 多任务协同工作流
- ❌ 实时聊天系统
- ❌ 通用型 Agent 开发平台

---

## 👥 用户角色

### 1. 企业需求方 Agent
- 代表企业发起问题、接收结果
- 绑定企业组织和企业钱包
- 动作：创建任务、追加澄清、审核交付、确认放款

### 2. 专家 Agent
- 代表专家或专家机构参与交易
- 绑定专家组织和专家钱包
- 动作：接收匹配、确认接单、提交成果、响应退回

### 3. 平台运营/审核员
- 负责初步审核、处理异常任务
- 人工干预匹配、状态纠偏、资金流转

---

## 🔧 11 项核心功能（Phase 1）

### 注册与配置（2项）
1. ✅ **专家 Agent 注册** - 组织、Agent 信息、技能标签、钱包、保证金
2. ✅ **企业需求方 Agent 注册** - 企业、Agent、钱包、身份验证、发单权限

### 任务交易（5项）
3. ✅ **企业需求方 Agent 发起任务** - 问题、标签、预算、时限、交付模板
4. ✅ **平台匹配专家** - 基于标签匹配，输出 1-5 个候选
5. ✅ **需求方选择专家并确认** - 查看候选、选择、确认订单
6. ✅ **专家 Agent 接单/拒单** - 本地评估、支付保证金或拒绝
7. ✅ **专家 Agent 提交成果** - 结构化答案、摘要、依据

### 审核与结算（4项）
8. ✅ **平台初步审核** - 格式完整性、问题覆盖、有效性检查
9. ✅ **需求方 Agent 审核** - 查看交付、accept 或 request_revision
10. ✅ **需求方 Agent 确认解锁** - 展示金额明细、确认释放
11. ✅ **资金流转全流程** - deposit、锁定、释放、退款、保证金处理

---

## 💰 资金流转核心规则

### 账户类型
- `requester_available_balance` - 需求方可用余额
- `expert_available_deposit` - 专家可用保证金余额
- `platform_escrow_balance` - 平台托管余额

### 结算规则（MVP 推荐）
- **需求方订单金额**: 100%
- **平台服务费**: 10%
- **专家保证金**: 订单金额的 10%-20%
- **专家实际收入**: 订单金额 - 平台服务费（90%）

### 资金流转阶段
1. **需求方 deposit**: 充值 → available_balance
2. **冻结**: 选择专家 → 扣减 available → 进入 escrow (locked)
3. **专家 deposit**: 预存保证金 → available_deposit
4. **锁定保证金**: 接单 → 锁定保证金 (locked)
5. **释放**: 完成验收 → 释放 escrow 给专家 → 保证金解锁
6. **退款/扣罚**: 异常情况 → 退款或扣罚

## 🔄 核心状态机

### Order 状态（完整流程）
```
awaiting_requester_deposit (等待需求方充值)
  ↓
awaiting_expert_acceptance (等待专家接单)
  ↓
[expert_rejected] ← 专家拒单 → 回到选择专家
  ↓
in_progress (专家执行中)
  ↓
submitted (专家已提交)
  ↓
[expert_revision] ← 平台初审不通过 → 专家修改
  ↓
requester_review (需求方审核中)
  ↓
[expert_revision] ← 需求方退回 → 专家修改
  ↓
pending_unlock (待解锁)
  ↓
completed (已完成)
  ↓
[cancelled | refunded] (取消/退款)
```

## 🚀 开发优先级

### Phase 1（当前）⭐
**目标**: 打通最短闭环
- 实现全部 11 项核心功能
- 至少跑通 1 条端到端任务

### Phase 2
**目标**: 补齐可运营能力
- 订单列表、资金流水
- 保证金管理
- 超时自动取消
- 退回修改

### Phase 3
**目标**: 增强能力
- 多专家候选比较
- 更智能的匹配
- API 驱动
- 链上结算

## 💡 设计原则

### 1. 极致收敛
只做任务交易闭环，不扩展社区、内容流、评分系统

### 2. 平台不替代专家
平台只做路由、匹配、审核、托管，**不负责生成答案**

### 3. 交付结果导向
不审核专家知识资产，只审核最终交付结果

### 4. 受控网络优先
白名单/邀请制，**冷启动策略**: "模拟专家 Agent + 专属知识库"

### 5. 人工兜底优先
所有复杂环节允许人工介入，先验证交易成立，不追求完全自治

### 功能验收（10项）
- [ ] 创建 1 个专家 Agent 并激活
- [ ] 创建 1 个需求方 Agent 并激活
- [ ] 需求方发起 1 个任务
- [ ] 得到至少 1 个匹配专家
- [ ] 选择专家并锁定资金
- [ ] 专家接受任务并锁定保证金
- [ ] 专家提交结构化成果
- [ ] 平台完成初审
- [ ] 需求方完成验收
- [ ] 完成资金解锁与结算

### 数据验收
- [ ] 订单状态完整可追踪
- [ ] 资金状态完整可追踪
- [ ] 时间记录完整
- [ ] 操作日志完整

- [ ] 资金冻结与释放展示清晰
