## P1-Landing: 着陆页

### 页面目标
- 展示 Neon Protocol 核心价值
- 引导用户注册或登录
- 快速让用户理解产品

### 布局结构

**顶部导航栏**：
- Logo（左）
- 导航菜单（中）: 产品介绍 | 专家网络 | 定价
- CTA 按钮（右）: 登录 | 注册

**Hero 区域**：
- 主标题: "Find the Right Manufacturing Experts, Instantly"
- 副标题: "The First Agent-to-Agent Protocol for Professional Knowledge Trading"
- 核心输入框: "What problem are you trying to solve?"
- 示例文本: "We need to evaluate [chip model] supply risk..."
- 提交按钮: "Submit Request →"

**信任元素区域**：
- 统计数字: "30,000+ Expert Interviews" | "500+ Companies" | "24hr Response"
- Logo 墙: 合作伙伴/客户 Logo（可选）

**功能介绍区域**（3栏）：
- 卡片1: "Smart Matching" - AI 驱动的专家匹配
- 卡片2: "Secure Delivery" - 本地 Agent，保护 IP
- 卡片3: "Fast Settlement" - USDC 即时结算

**底部**：
- Footer: 关于 | 联系 | 条款 | 隐私

### 核心组件
1. **Hero 输入框** - 大型文本框，支持 Markdown 预览
2. **统计卡片** - 数字 + 说明
3. **功能卡片** - 图标 + 标题 + 描述

### 数据需求
- 统计数字（静态）
- 合作伙伴 Logo（静态）

### 交互逻辑
1. 用户在输入框输入问题
2. 点击 "Submit Request"
3. **未登录**: 跳转到 P2-Login
4. **已登录**: 跳转到 P8-Task-Create，自动填充问题

### 跳转关系
- → P2-Login（点击"登录"）
- → P3-Register（点击"注册"）
- → P8-Task-Create（提交问题后，已登录）

## P2-Login: 登录页

### 页面目标
- 用户身份验证
- 快速进入系统

### 布局结构

**左侧区域**（装饰）：
- 大图或插图
- 品牌标语

**右侧区域**（表单）：
- 标题: "Welcome Back"
- 邮箱输入框
- 密码输入框
- "记住我"复选框
- "登录"按钮
- "忘记密码？"链接
- "没有账号？注册"链接

### 核心组件
- **登录表单**: 邮箱 + 密码 + 提交
- **错误提示**: 登录失败时显示

### 数据需求
- 邮箱
- 密码

### 交互逻辑
1. 用户输入邮箱和密码
2. 点击"登录"
3. **成功**: 跳转到对应的 Dashboard
   - 需求方 → P7-Requester-Dashboard
   - 专家 → P14-Expert-Dashboard
   - 管理员 → P21-Admin-Dashboard
4. **失败**: 显示错误提示（"邮箱或密码错误"）

### 跳转关系
- → P7-Requester-Dashboard（需求方登录成功）
- → P14-Expert-Dashboard（专家登录成功）
- → P21-Admin-Dashboard（管理员登录成功）
- → P3-Register（点击"注册"）
- → P1-Landing（点击 Logo）

## P4-Requester-Register: 需求方注册

### 页面目标
- 创建需求方组织账户
- 收集必要的企业信息

### 布局结构

**进度指示器**：
- Step 1 of 2: Account Info

**表单区域**：

**Step 1: 组织信息**
- 企业名称（必填）
- 企业邮箱（必填）
- 行业类型（下拉选择）: 半导体 | 汽车制造 | 电子设备 | 其他
- 公司规模（下拉）: 1-50 | 51-200 | 201-500 | 500+
- 公司网站（选填）

**Step 2: Agent 信息**
- Agent 名称（必填）
- 常用问题类型（多选）: 供应链 | 工艺工程 | 市场准入 | 合规 | 其他
- 任务通知邮箱（必填）
- 支付钱包地址（必填）

**按钮区域**：
- "上一步"（Step 2）
- "创建账户"（Step 2）
- "下一步"（Step 1）

### 核心组件
- **多步骤表单**: Step 1 + Step 2
- **进度条**: 显示当前步骤
- **下拉选择器**: 行业类型、公司规模
- **多选标签**: 常用问题类型

### 数据需求
- 企业名称、邮箱、行业、规模、网站
- Agent 名称、问题类型、通知邮箱、钱包地址

### 交互逻辑
1. 填写 Step 1 信息
2. 点击"下一步"
3. 填写 Step 2 信息
4. 点击"创建账户"
5. **成功**: 跳转到 P7-Requester-Dashboard
6. **失败**: 显示错误提示

### 跳转关系
- → P7-Requester-Dashboard（注册成功）
- → P3-Register（返回）

## P5-Expert-Register: 专家注册

### 页面目标
- 创建专家组织账户
- 收集专家资质信息

### 布局结构

**进度指示器**：
- Step 1 of 3: Account Info

**表单区域**：

**Step 1: 组织信息**
- 专家/机构名称（必填）
- 联系邮箱（必填）
- 机构类型（下拉）: 独立专家 | 咨询公司 | 研究机构
- 公司网站（选填）

**Step 2: Agent 信息**
- Agent 名称（必填）
- 专业领域标签（多选）: 半导体 | 汽车制造 | 新能源 | 工业自动化 | 其他
- 服务地区（多选）: 中国 | 日本 | 韩国 | 东南亚 | 欧美
- 擅长问题类型（多选）: 供应链 | 工艺工程 | 市场准入 | 合规 | 其他
- 默认交付时限（下拉）: 24小时 | 48小时 | 72小时 | 1周
- Agent 回调 URL（选填）

**Step 3: 钱包与保证金**
- 收款钱包地址（必填）
- 保证金充值（显示最低保证金要求）: 最低 $500
- 充值方式说明（MVP: 联系客服手动充值）

**按钮区域**：
- "上一步"
- "下一步" / "提交审核"

### 核心组件
- **多步骤表单**: Step 1-3
- **标签选择器**: 多选标签（专业领域、服务地区等）
- **保证金提示**: 显示最低要求

### 数据需求
- 组织名称、邮箱、类型、网站
- Agent 名称、技能标签、服务地区、问题类型、交付时限、回调 URL
- 钱包地址

### 交互逻辑
1. 依次填写 3 个步骤
2. 点击"提交审核"
3. **成功**: 跳转到 P16-Agent-Detail（显示"待审核"状态）
4. 平台审核通过后，Agent 变为 active

### 跳转关系
- → P16-Agent-Detail（提交成功）
- → P3-Register（返回）

## P8-Task-Create: 创建任务

### 页面目标
- 需求方提交新任务
- 收集完整的需求信息

### 布局结构

**页面标题**：
- "创建新任务"

**表单区域**（分为多个部分）：

**Part 1: 问题描述**
- 任务标题（必填）: 文本框
- 详细描述（必填）: 多行文本框，支持 Markdown
- 附件上传（选填）: 上传相关文档（PDF/Word）

**Part 2: 标签与分类**
- 行业标签（多选）: 芯片半导体 | 汽车制造 | 消费电子 | 工业设备
- 地区标签（多选）: 中国 | 日本 | 韩国 | 东南亚 | 欧美
- 问题类型（多选）: 供应链 | 工艺工程 | 市场准入 | 合规 | 成本分析

**Part 3: 预算与时间**
- 预算范围（必填）: 数字输入框 + 货币选择（USDC）
- 期望交付时间（必填）: 日期选择器
- 紧急程度（单选）: 普通 | 加急（24小时内）

**Part 4: 交付要求**
- 交付模板（下拉选择）:
  - 标准问答模板（推荐）
  - 风险评估模板
  - 可行性分析模板
  - 自定义模板
- 是否需要引用依据（复选框）

**按钮区域**：
- "保存草稿"
- "提交并开始匹配"（主按钮）

### 核心组件
- **Markdown 编辑器**: 详细描述
- **标签选择器**: 多选标签组件
- **日期选择器**: 交付时间
- **文件上传**: 附件上传

### 数据需求
- 标题、描述、附件
- 行业标签、地区标签、问题类型
- 预算、交付时间、紧急程度
- 交付模板、是否需要依据

### 交互逻辑
1. 填写表单
2. 点击"保存草稿" → 任务保存为 draft 状态
3. 点击"提交并开始匹配" → 任务创建并触发匹配
4. **成功**: 跳转到 P10-Match-Results

### 跳转关系
- → P10-Match-Results（提交成功）
- → P7-Requester-Dashboard（取消）

## 🎨 通用组件库

### 1. 状态徽章（Status Badge）
- **颜色编码**:
  - 绿色: 成功/完成/active
  - 蓝色: 进行中/pending
  - 黄色: 等待/警告
  - 红色: 失败/取消/rejected
  - 灰色: 草稿/draft

### 2. 信息卡片（Info Card）
- **结构**: 标题 + 内容 + 操作按钮
- **用途**: 任务信息、专家信息、资金详情

### 3. 标签选择器（Tag Selector）
- **功能**: 多选标签，带搜索
- **用途**: 技能标签、地区标签

### 4. 时间线（Timeline）
- **功能**: 显示订单进度历史
- **结构**: 时间点 + 事件描述

### 5. 表格列表（Table List）
- **功能**: 展示任务/订单列表
- **列**: 名称 | 状态 | 时间 | 操作

### 6. 表单向导（Form Wizard）
- **功能**: 多步骤表单
- **组件**: 步骤指示器 + 表单区域 + 导航按钮
