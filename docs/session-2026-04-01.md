### Pitch Deck 分析

**已收到材料**：

- ✅ 英文演示稿——Pitch_Deck.pdf（5个版本）

### 核心理念确认

**产品定位**：
- **名称**: Neon Protocol
- **定位**: 全球首个 Agent 技能结果交易与流动性网络
- **核心价值**: 交易结果，不交易过程（保护专家 IP）

**关键痛点**：
1. 专家恐惧 IP 泄露（不敢上云）
2. 传统专家网络效率低（GLG/凯盛：3天 + 上万美金）
3. 通用大模型缺乏行业深度私有数据

**解决方案**：
1. 本地 Agent 外部驱动
2. 只交易结果，不暴露过程
3. Web3 原生结算（USDC 智能合约）

## 2. MVP 场景：半导体供应链极速排雷

### 场景描述

**背景**: 新能源车企芯片供应链危机（某国出口管制）

**传统方式**：
- 时间：3天
- 成本：上万美金
- 流程：销售对齐 → 盲筛专家 → 电话会议

**Neon Protocol**：
- 时间：2小时
- 成本：2,500 USDC
- 流程：需求输入 → AI澄清 → Agent匹配 → 专家确认 → 自动结算

### 用户旅程（5个核心页面）

1. **需求输入页** - 车企输入大白话需求
2. **AI澄清页** - Clarification Agent 确认细节
3. **Agent市场** - 展示匹配的专家 Agent（98%匹配度）
4. **专家控制台** - 专家审核 Agent 生成的报告
5. **结算完成页** - 智能合约自动分账

## 3. 核心产品决策 ⭐

### 决策 1: Agent 实现方式
- **技术方案**: OpenClaw + 专家访谈知识库
- **原理**: OpenClaw 检索知识库 → 生成答案
- **扩展性**: 其他专家可用不同技术栈

### 决策 2: 平台边界（参考 Moltbook）
- **平台提供**: API 注册 + A2A 通信 + 智能合约结算
- **平台不提供**: Agent 搭建、算力、托管
- **专家自由**: 自行选择技术栈（OpenClaw/LangChain/AutoGPT/自研）

### 决策 3: 质量审核机制
- **平台负责**: 基础身份审核、投诉处理
- **专家负责**: 确认 Agent 输出、维护知识库
- **需求方负责**: 评估答案质量、决定是否解锁

### 决策 4: 定价模式
- **初期**: 标准定价（$1,000 - $5,000 / 任务）
- **暂不**: 浮动定价、拍卖机制

### 决策 5: 知识产权
- **初期**: 暂不考虑 IP 保护机制
- **未来**: 可能加入零知识证明

### 决策 6: Agent 维护
- **平台负责**: 自己的模拟 Agent（30,000+ 访谈数据）
- **专家负责**: 第三方 Agent 的维护、运行、算力

## 5. Demo 开发方案

### 方案 A: 极速原型（推荐）
- **技术栈**: Next.js 15 + Tailwind + Framer Motion
- **特点**: 纯前端，无后端，完全模拟交互
- **时间**: 4-6 小时
- **适用**: 投资人演示、客户展示

### 方案 B: 半真实系统
- **技术栈**: Next.js + Prisma + SQLite + Wagmi
- **特点**: Mock 数据库，真实匹配逻辑
- **时间**: 3-5 天
- **适用**: 内部验证

### 方案 C: 完整 MVP
- **技术栈**: Next.js + PostgreSQL + 智能合约
- **特点**: 可实际运行
- **时间**: 2-4 周
- **适用**: 正式上线
